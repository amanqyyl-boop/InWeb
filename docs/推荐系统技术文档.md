## 1. 系统概述

推荐系统位于 `backend/cnn_recommender.py`，是一个**纯 Python 实现**的混合推荐引擎，核心依赖仅为 **scikit-learn** 和 **NumPy**。

### 核心思想

- **聚类**：用 KMeans 将小说按特征自动分组，发现隐含的类别结构
- **相似度**：用 Cosine Similarity 度量用户偏好与小说特征的匹配程度
- **混合排序**：聚类匹配 + 内容相似度 + 用户偏好 + 基础质量 加权融合

### 依赖

```text
scikit-learn >= 1.4.0
numpy >= 1.24.0
```

## 2. 算法选型

### 为什么选 KMeans + Cosine Similarity？

| 算法 | 优点 | 缺点 | 本项目适用性 |
|------|------|------|-------------|
| **KMeans 聚类** | 无监督，自动发现结构；速度快 | 需指定 K 值；对异常值敏感 | ✅ 小说分类清晰，K=8 效果良好 |
| **Cosine Similarity** | 不受向量长度影响；直观易懂 | 忽略数值尺度 | ✅ 配合 StandardScaler 使用 |
| 矩阵分解 (SVD) | 捕捉隐式特征 | 数据稀疏时效果差 | ❌ 用户-小说交互矩阵极稀疏 |
| 协同过滤 | 利用群体智慧 | 冷启动问题严重 | ❌ 新用户无历史行为 |
| 深度学习 | 表达能力强 | 需要大量数据和 GPU | ❌ 违反复杂度的设计目标 |

### 最终方案：混合推荐

```
聚类匹配(35%) + 内容相似度(30%) + 注册偏好(25%) + 基础质量(10%)
```

四者加权融合，兼顾了**群体模式、个性化匹配、用户主动选择、内容质量**。

## 3. 特征工程

每本小说被编码为一个 **12 维特征向量**：

```
[玄幻, 仙侠, 都市, 言情, 历史, 科幻, 悬疑, 幻想, 军事, 网游, 评分, 热度]
 ├── 0/1 one-hot ──┤                                     ├── 归一化值 ──┤
       10 维分类特征                                          2 维数值特征
```

### 特征构建代码

```python
def _build_features(self, novels):
    n = len(novels)
    features = np.zeros((n, NUM_CATEGORIES + 2))

    for i, novel in enumerate(novels):
        cat_idx = CAT_TO_IDX.get(novel['category'], -1)
        if cat_idx >= 0:
            features[i, cat_idx] = 1.0               # one-hot 分类
        features[i, NUM_CATEGORIES]     = novel.get('rating', 5) / 10.0      # 评分 [0,1]
        features[i, NUM_CATEGORIES + 1] = novel.get('hot', 0) / 15000.0      # 热度 [0,1]

    return features
```

### StandardScaler 标准化

在聚类前，特征经过 `StandardScaler` 标准化：

```python
self.scaler = StandardScaler()
features_scaled = self.scaler.fit_transform(features)
```

标准化使每个特征的均值为 0、方差为 1，避免"热度"这类数值大的特征主导聚类结果。

## 4. KMeans 聚类

### 原理

KMeans 将 N 个小说样本划分到 K 个聚类中，使得每个样本到其所属聚类中心的距离平方和最小：

$$\min \sum_{i=1}^{K} \sum_{x \in C_i} \|x - \mu_i\|^2$$

### 参数配置

```python
self.kmeans = KMeans(
    n_clusters=8,        # 将小说分为 8 个簇
    random_state=42,     # 固定随机种子，保证可复现
    n_init='auto'        # 自动选择初始化次数
)
```

`n_clusters=8` 的选择依据：
- 系统中实际有 10 个分类，但部分分类（如言情）小说数量极少
- 8 个簇既能捕捉主要分类结构，又不会过度细分
- 聚类结果自动发现"玄幻+仙侠"相近、"历史+军事"相近等模式

### 聚类效果示意

```
簇 0: [圣墟, 完美世界, 遮天, 深空彼岸, ...]    ← 仙侠类为主
簇 1: [斗破苍穹, 沧元图, ...]                    ← 玄幻类为主
簇 2: [临高启明, 上品寒士, ...]                  ← 历史类为主
簇 3: [全职高手, 高手寂寞, ...]                  ← 网游类为主
...
```

## 5. Cosine Similarity 相似度计算

### 用户偏好向量

从用户的浏览历史构建偏好向量：

```python
if browsed_vectors:
    user_vec = np.mean(browsed_vectors, axis=0)     # 浏览过的小说特征取平均
else:
    # 无历史时用注册偏好构造虚拟向量
    raw[cat_idx] = 1.0
    user_vec = self.scaler.transform(raw.reshape(1, -1))[0]
```

### 两步相似度计算

**① 用户 vs 聚类中心** — 找到用户偏好的聚类

```python
sims = cosine_similarity(user_vec.reshape(1, -1), self._cluster_centers)[0]
```

$$\text{cosine\_sim}(A, B) = \frac{A \cdot B}{\|A\| \|B\|}$$

**② 用户 vs 每本小说** — 在偏好聚类中精确排序

```python
sim = cosine_similarity(user_vec.reshape(1, -1), novel_vec)[0, 0]
```

## 6. 混合评分策略

### 评分公式

$$\text{score} = \underbrace{C_{\text{cluster}} \times 0.35}_{\text{聚类匹配}} + \underbrace{C_{\text{sim}} \times 0.30}_{\text{内容相似度}} + \underbrace{B_{\text{pref}} \times 0.25}_{\text{注册偏好}} + \underbrace{Q_{\text{base}} \times 0.10}_{\text{基础质量}}$$

### 各项含义

| 分量 | 来源 | 权重 | 说明 |
|------|------|------|------|
| **聚类匹配分** | 用户向量到各聚类中心的 cosine 相似度 | **35%** | 用户整体偏好哪个小说群组 |
| **内容相似度** | 用户向量 vs 单本小说的 cosine 相似度 | **30%** | 这本小说和用户有多"像" |
| **注册偏好偏置** | 用户注册时选择的分类 | **25%** | 用户主动表达的偏好 |
| **基础质量** | 评分/10 + 热度/20000 | **10%** | 热门和高分作品的基础加分 |

### 降权处理

已浏览过的小说减 0.3 分，避免重复推荐：

```python
if novel['id'] in browsed_ids:
    total -= 0.3
```

## 7. 在线学习机制

### 策略：懒更新（Lazy Update）

KMeans 本身不支持严格的在线增量学习，因此采用**累计触发**策略：

```python
def learn_from_browse(self, novel_id, all_novels, browse_records):
    unique_ids = {r['novel_id'] for r in browse_records}
    if len(unique_ids) % 10 == 1 and len(unique_ids) > 1:
        self.fit(all_novels)    # 重新聚类
```

- 用户每浏览一本小说，触发一次 `learn_from_browse()`
- 但**只有浏览了 10 本不同的书后**，才会真正重新执行 KMeans 聚类
- 这样避免了频繁重聚类的计算开销，同时保证了推荐结果随用户行为演变

### 冷启动处理

新用户没有任何浏览历史时：

1. 用注册时选择的偏好分类构造虚拟特征向量
2. 直接使用注册偏好偏置（权重 25%）产生初始推荐
3. 用户浏览第一本书后开始建立真实偏好

## 8. 完整数据流

```
┌─────────────────────────────────────────────────────────┐
│                   启动时                                  │
│  scan_novel_library() → ALL_NOVELS                      │
│         ↓                                               │
│  首次推荐请求时:                                         │
│  SKRecommender.fit(ALL_NOVELS)                          │
│    → _build_features() → 12维特征矩阵                   │
│    → StandardScaler.fit_transform()                     │
│    → KMeans.fit() → 8个聚类 + 聚类中心                  │
│    → 缓存 _features_scaled, _cluster_labels             │
├─────────────────────────────────────────────────────────┤
│                   用户请求推荐                            │
│  GET /api/recommend/{uid}                               │
│         ↓                                               │
│  recommend_novels()                                     │
│    → 读取浏览历史 browse_records                         │
│    → 构建用户偏好向量 user_vec                           │
│    → cosine_similarity(user_vec, 聚类中心) → 聚类权重    │
│    → 遍历 ALL_NOVELS:                                   │
│        聚类匹配分(35%) + 内容相似度(30%)                  │
│      + 注册偏好(25%) + 基础质量(10%)                     │
│    → 排序取 top-12 返回                                  │
├─────────────────────────────────────────────────────────┤
│                   用户浏览小说                            │
│  POST /api/browse → learn_from_browse()                 │
│         ↓                                               │
│  浏览满 10 本不同书?                                     │
│    → 是: SKRecommender.fit() 重新聚类                    │
│    → 否: 跳过                                           │
└─────────────────────────────────────────────────────────┘
```

## 10. 与旧版对比

| 维度 | 旧版（纯 NumPy CNN） | 新版（scikit-learn） |
|------|--------------------|--------------------|
| **输入特征** | 仅分类序号序列 | 12 维特征向量（分类+评分+热度） |
| **模型** | 手写 1D CNN（Embedding → Conv → Pool → Dense） | KMeans 聚类 + Cosine Similarity |
| **参数数量** | ~500 个（Embedding + Conv + Dense） | 聚类中心 8×12 = 96 个 |
| **训练方式** | 逐个样本 online update（梯度下降） | Batch fit（整表聚类） |
| **可解释性** | 低（CNN 内部表示难以理解） | 高（聚类中心和相似度直观可解释） |
| **框架依赖** | 无（纯 NumPy） | scikit-learn + NumPy |
| **推理速度** | 快（~1ms） | 快（~2ms，含相似度计算） |
| **冷启动** | 仅用注册偏好 | 注册偏好 + 聚类先验 |
| **代码行数** | ~280 行 | ~180 行 |

### 为什么新版更好？

1. **真正的 ML 框架** — scikit-learn 是经过工业验证的机器学习库，算法实现可靠
2. **特征更丰富** — 不仅看分类，还融合了评分和热度信息
3. **可解释性强** — 可以明确知道"这本书推荐给你是因为它和你看过的书属于同一个聚类"
4. **代码更简洁** — 减少了 100 行手写神经网络代码，维护成本更低
5. **标准化处理** — StandardScaler 保证了特征之间的公平比较
